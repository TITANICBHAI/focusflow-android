3단계 강제 시스템

대부분의 차단기는 한 개의 레이어를 사용하지만, FocusFlow는 세 개를 쌓습니다. 각 레이어는 이전 레이어가 남긴 우회 방법을 닫습니다.

1

접근성 서비스

모든 앱 창을 모니터링합니다. 차단된 앱 감지 후 밀리초 내에 실행 — 300ms 간격으로 최대 5번 재검사. 활성 세션 중 Play 스토어 설치, 제거 다이얼로그, 접근성 설정으로의 탐색도 차단합니다.

2

로컬 널 라우팅 VPN

차단된 앱 트래픽을 패킷을 전달하지 않는 로컬 VPN 터널로 라우팅합니다. 앱별 모드: 차단된 앱만 인터넷을 잃습니다. 전체 모드: 긴급 통화를 제외한 모든 인터넷 차단. 어떤 트래픽도 기기를 벗어나지 않습니다.

3

기기 관리자

Android 설정을 통한 제거를 차단합니다. System Guard는 세션 중 데이터 지우기 및 접근성 화면으로의 탐색을 차단합니다. 기본적으로 저장된 SHA-256 해시 세션 PIN — 손상된 JS 브리지도 올바른 PIN 없이는 세션을 종료할 수 없습니다.

내용물

다른 차단기에는 없는 기능

이것들은 점진적인 개선이 아닙니다. Freedom, Opal, AppBlock 또는 Digital Wellbeing에서는 찾을 수 없는 기능들입니다.

📊

유혹 로그 + 주간 보고서

타임스탬프와 함께 차단된 앱의 모든 시도를 기록합니다 (최대 500개). 매주 일요일 오전 8시에 앱별 7일 행동 요약이 푸시 알림으로 전달됩니다. 타이머가 아닌 패턴을 파악합니다.

📅

예약된 그레이아웃 창

활성 세션 없이 월~금 09:00~18:00에 Instagram을 차단합니다. 앱별, 요일별 캘린더 차단. 4가지 타이밍 모드와 독립적으로 — 접근성 서비스에 의해 직접 강제됩니다.

📱

콘텐츠별 차단

YouTube Shorts 차단기는 YouTube를 완전히 접근 가능한 상태로 유지하면서 Shorts 플레이어를 닫습니다. Instagram Reels 차단기도 마찬가지 방식으로 작동합니다. 앱 전체가 아닌 중독성 루프를 차단합니다.

⚡

혐오 피드백

화면이 2% 하드웨어 밝기로 어두워짐 + 70% 검은 오버레이 + 진동 펄스 루프 + 알림음 — 차단된 앱 감지 시 모두 동시에 발동. 행동뿐만 아니라 반사신경을 조건화합니다.

🏠

런처 교체

FocusFlow를 홈 화면으로 설정합니다. HOME 버튼은 최소한의 그리드에 화이트리스트 앱만 표시합니다. 앱 서랍도 차단 목록을 존중합니다. 홈 화면이 강제 실행자가 됩니다.

🔄

부팅 복구

BootReceiver는 활성 세션 중 기기 재시작 후 접근성 서비스와 VPN을 자동으로 재시작합니다. 시계 변조 감지가 부팅 시 세션 타임스탬프를 검증합니다 — "시스템 시계 변경" 우회는 작동하지 않습니다.

📦

사이드로드 가드

PackageInstallReceiver가 ACTION_PACKAGE_ADDED를 모니터링합니다. 세션 중 설치된 앱은 자동으로 차단 목록에 추가되고 혐오 진동 피드백을 트리거합니다. "우회하기 위해 다른 브라우저 설치" 허점이 닫혔습니다.

⏱

일일 허용량 엔진

앱당 3가지 하위 모드: 횟수 (일일 열기 횟수), 시간 예산 (일일 분), 간격 (사용 간 쿨다운). 단순한 세션 차단을 넘어선 세밀한 제어.

시작하기

몇 분 안에 시작

두 가지 권한을 부여하면 보호됩니다. FocusFlow가 나머지 모든 것을 백그라운드에서 처리합니다.

1

권한 부여

접근성 서비스 및 사용 통계 액세스를 활성화합니다 — OS 수준에서 앱을 감지하고 차단하는 데 두 가지 모두 필요합니다.

2

모드 선택

작업 기반, 독립형, 항상 켜짐 또는 예약된 그레이아웃. 차단하려는 앱을 추가합니다 (또는 허용 목록 화이트리스트).

3

차단 시작

FocusFlow가 3가지 강제 레이어를 모두 활성화합니다. 지속적인 알림이 세션 상태와 제어를 표시합니다.

4

데이터 확인

유혹 로그를 확인합니다. 일요일 보고서를 읽습니다. 어떤 앱이 주의를 가장 많이 끄는지 확인하고 차단 목록을 조정합니다.

질문

자주 묻는 질문

설치 전 알아야 할 모든 것.

FocusFlow는 루팅이 필요한가요?
아니요. FocusFlow는 표준 Android 권한 시스템 내에서 완전히 작동합니다 — 접근성 서비스, VPN 및 기기 관리자. 루팅이나 시스템 수정이 필요하지 않습니다. 30개 이상의 OEM 브랜드의 스톡 Android 8.0 이상에서 실행됩니다.
FocusFlow는 어떻게 제거를 방지하나요?
세 가지 레이어: (1) Android 기기 관리자 모드가 일반 설정 → 앱 → 제거 경로를 차단; (2) System Guard가 활성 세션 중 접근성 설정, 데이터 지우기 및 제거로의 탐색을 차단; (3) 기본적으로 저장된 SHA-256 해시 세션 PIN은 손상된 JavaScript 레이어도 올바른 PIN 없이 세션을 종료할 수 없음을 의미합니다.
유혹 로그와 주간 보고서란 무엇인가요?
유혹 로그는 차단된 앱의 모든 시도를 기록합니다 — 앱 이름, 패키지 이름, 타임스탬프 — 최대 500개. 매주 일요일 오전 8시에 FocusFlow가 앱별 7일 합계를 포함한 푸시 알림을 전달합니다 (예: "Instagram: 18회 시도, TikTok: 이번 주 14회 시도"). 주간 행동 보고서를 제공하는 스크린 타임 앱은 다른 곳에 없습니다.
FocusFlow는 YouTube 전체를 차단하지 않고 YouTube Shorts를 차단할 수 있나요?
네. YouTube Shorts 차단기는 메인 피드, 검색, 구독 및 동영상 재생을 완전히 접근 가능하게 유지하면서 YouTube 내 Shorts 플레이어를 닫습니다. Instagram Reels 차단도 동일하게 작동합니다 — Reels 피드가 차단되고 나머지 Instagram은 그대로입니다.
세션 중에 새 앱을 설치하면 어떻게 되나요?
PackageInstallReceiver가 ACTION_PACKAGE_ADDED 브로드캐스트를 모니터링합니다. 새로 설치된 앱은 자동으로 차단 목록에 추가되고 혐오 진동 피드백을 트리거합니다. 일반적인 "세션 중 경쟁 브라우저 설치" 허점이 닫혔습니다.
FocusFlow는 데이터를 수집하거나 공유하나요?
아니요. 로컬 VPN은 외부 서버로 패킷을 전달하지 않습니다. 유혹 로그, 작업 데이터 및 모든 차단 상태는 기기의 Android SharedPreferences 및 SQLite에 저장됩니다. FocusFlow는 서버에 원격 측정, 분석 또는 개인 데이터를 전송하지 않습니다.
전화 재부팅 후에도 FocusFlow가 작동하나요?
네. BootReceiver는 활성 세션 중 기기 재시작 후 접근성 서비스와 VPN을 자동으로 재시작합니다. 시계 변조 감지도 부팅 시 세션 타임스탬프를 검증합니다 — "타이머를 만료시키기 위해 시스템 시계 변경" 우회는 효과가 없습니다.
FocusFlow는 어떤 Android 기기를 지원하나요?
30개 이상의 Android OEM 브랜드: Samsung (OneUI), Xiaomi (MIUI/HyperOS), Oppo (ColorOS), Realme, Vivo (Funtouch/OriginOS), OnePlus (OxygenOS), Huawei (EMUI/HarmonyOS), Honor, Motorola, Asus (ZenUI/ROG), Nothing OS, Nokia/HMD, Sony Xperia, Meizu (Flyme), LG, Lenovo, TCL/Alcatel, ZTE, Wiko, Black Shark 및 트랜시온 브랜드 (Infinix, Tecno, itel).
FocusFlow는 정말 무료인가요?
네, 완전히 무료입니다. 구독, 앱 내 구매, 프리미엄 등급이 없습니다. 모든 기능 — 접근성 서비스 강제, VPN 차단, 기기 관리자, 유혹 로그, 주간 보고서, 예약된 그레이아웃 창, 스케줄링 엔진 — 이 무료로 사용 가능합니다.
